## 3. Audit Technique

### 3.1 Indexation et crawl

| Élément | État | Notes |
|---|---|---|
| `robots.txt` | OK | Autorise tous les crawlers IA (GPTBot, ClaudeBot, PerplexityBot, etc.) |
| Sitemap | OK | Généré par `@astrojs/sitemap`, auto-découvrable |
| RSS | OK | `rss.xml.ts` avec auto-discovery dans `<head>` |
| `llms.txt` | OK | Index structuré de tous les articles (~23KB) |
| `_redirects` | OK | 25+ redirects 301 configurés (WP → Astro) |
| Canonical | OK | `<link rel="canonical">` sur toutes les pages |
| `noindex` tags | OK | Aucune page tag n'a de `noindex` (à considérer pour les tags à 1 article) |

### 3.2 Sécurité et headers HTTP

| Élément | État | Fichier |
|---|---|---|
| HSTS | Présent | `public/_headers` |
| CSP | Présent | `public/_headers` |
| X-Frame-Options | Présent | `public/_headers` |
| Referrer-Policy | Présent | `public/_headers` |
| Permissions-Policy | Présent | `public/_headers` |

> Vérification runtime nécessaire pour confirmer que Cloudflare Pages sert bien ces headers.

Non disponibles via le MCP GSC (requiert l'API Search Console v3 avec `query.pageSpeed`).
**Recommandation** : Vérifier manuellement dans la console GSC > Expérience > Core Web Vitals, ou utiliser PageSpeed Insights.
